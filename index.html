Attribute

  • Simple

    Matches an element that contains a certain attribute, regardless of the attribute's value.

    input[type] {font-weight:bold;}
    p[id] {background:#eee; border:1px solid #666;}

  • Exact Value

    Selects an element on the precise and complete value of an attribute.

    input[type="text"] {font-weight:bold; background:#eee;}
    p[id="page-content"] {background:#eee; border:1px solid #666; box-shadow:5px 5px 5px #bbb;}

  • Partial Value

    p[class~="error"] {color:red; font-weight:bold;}
    a[title~="Recipe"] {background:#eee; font-weight:bold; color:maroon;}

  • Beginning Substring Value

    li[id^="nav"] {padding:0.5em; border:1px solid #ccc; box-shadow:0.1em 0.1em 0.1em #999;}

  • Ending Substring Value

    li[id$="-item"] {padding:0.5em; border:1px solid #ccc; box-shadow:0.1em 0.1em 0.1em #999;}

  • Arbitrary Substring Value

    menu li[id*="nav"] {padding:0.5em; border:1px solid #ccc; box-shadow:0.1em 0.1em 0.1em #999;}

  • Particular Attribute Selector(Language Selector)

    html[lang|="en-us"] {font-size:100%; font-style:normal;}
    p[lang|="bg"] {font-size:110%; font-style:italic;}
    img[href|="chart"] {border:2px solid #000; box-shadow:.2em .2em .2em #999;}

Pseudo-classes

Pseudo-classes are categorized as structural, negation and interaction ones.

Structural pseudo-classes

:empty
- matches an element that has no child nodes or content nodes (<p> </p> is not empty).

p:empty {padding: 1em; background: red;}
li:empty {display: none;}

:first-child
- matches an element that is the first child of another element.

li:first-child {border-radius:1 em 1em 0 0;}

:first-of-type
- matches an element that is the first child of its type of another element.

p:first-of-type {font-size:larger; line-height:150%; box-shadow:.2em .2em .2em #999;}

:lang
- applies to any element with associated language-encoding information. Language information cannot be assigned via CSS, it must be contained within or otherwise associated with the document.

html:lang(en) {background:#fff;}
p:lang(bg) {background:#eee; border: 1px solid #666;}

:last-child

li:last-child {border-radius:1 em 1em 0 0;}

:last-of-type

body > div:last-of-type {border-bottom:2px solid #999; margin-bottom:2em;}

:nth-child()
Matches every nth child, with the pattern of selection defined by the formula an+b. There also two keywords allowed - even (2n) and odd (2n+1).

li:nth-child(3n+1) {font-weight:bold;}
li:nth-child(4n-1) {font-weight:bold;} /*not the same as above*/
tbody tr:nth-child(odd) {background:#ccc;}

:nth-last-child()
This, too, matches every nth child following the an + b, pattern, but counting backwards from the last child. The even and odd keywords are allowed here, too.

tbody tr:nth-last-child(odd) {background:#ccc;}
li:nth-last-child(3n+1) {font-weight:bold;}

:nth-of-type()

body > p:nth-of-type(3n+1) {border:1px solid black;}

:nth-last-of-type()

td:nth-last-of-type(even) {background-color:#fcc;}

:only-child
- matches any element that is the only child of its parent.

ol li:only-child {background:#ccc; font-weight:bold;}

:only-of-type
- matches an element that is the only child of its type of its parent.

article section:only-of-type {border-bottom:2px solid gray; margin-bottom:2em;}

:root
- matches the root element in the document. In HTML, the root is always the html element. In XML, however, the root can be named randomly - hence the need for a generic root-element selector.

:root {font:medium serif;}

Negation pseudo-class

:not(e)
- matches every element that is not described by the simple selector e.

ul *:not(li) {text-indent: 2em;}
fieldset *:not([type="checkbox"]):not([type="radio"]) {margin:0 1em;}

Notes:
  • The "simple selector" restriction means that grouped and descendant selectors are not allowed within :not() expressions.
  • The simple selector cannot be the negation pseudo-class. In other words, you cannot write :not(:not(p)).
  • The negation pseudo-class can be chained with other pseudo-classes, as well as with instances of itself - see example above

Interaction pseudo-classes

:active

a:active {color:red;}
*:active {background:yellow;}

:checked

input:checked {outline: 3px solid #ccc;}
input[type="radio"]:checked {box-shadow: 1px 1px 5px blue;}

:disabled

input:disabled {opacity:0.5;}

:enabled

input:enabled {opacity:1; background:#eee;}

:focus

a:focus {outline: 1px dotted red;}
input:focus {background:yellow;}

:hover

a:hover {text-decoration:uderline;}
p:hover {background:yellow;}

:link

a:link {color:purple;}
*:link {text-decoration:none;}

:target
- matches an element which is itself matched by the fragment identifier portion of the URL (...#targetname).

:target {background: #ee0;}

:visited

a:visited {color: #333;}
*:visited {color:gray;}

Pseudo-elements

In CSS3, pseudo-elements are designated by double colons, to distinguish them from pseudo-classes. For historical reasons, browsers will support either syntax, but double colons are recommended.

::after
- contains generated content placed after the element's content.

.css-three::after {background: none repeat scroll 0 0 red; color: #fff; content: "3"; margin-left: 1em; opacity: 1; padding: 0 0.3em; border-radius:0.7em; font-weight:normal;}

::before
contains generated content placed before the element's content.

a[href]:before {content: "[LINK] ";}
p.note:before {content: "Note: "; font-weight:bold;}

::first-letter
- contains the first letter of an element.

h1::first-letter {font-size:larger;}
p::first-letter {text-decoration:underline;}

::first-line
- contains the first formatted line of an element, regardless of length.

p.lead::first-line {font-weight:bold;}

The ::first-line pseudo-element can be attached only to block-level elements.

The Cascade

The cascade can be defined as the sequence of steps CSS takes to resolve conflicting rules. It's the mechanism by which the user agent decides which rule to honor, whenever mutiple rules apply to the same element.
  1. Find all declarations that apply to the element and property in question, for the target media type. Declarations apply if the associated selector matches the element in question and the target medium matches the media list on all @media rules containing the declaration and on all links on the path through which the style sheet was reached.
  2. Sort according to importance (normal or important) and origin (author, user, or user agent). In ascending order of precedence:
    1. user agent declarations
    2. user normal declarations
    3. author normal declarations
    4. author important declarations
    5. user important declarations
  3. Sort rules with the same importance and origin by specificity of selector: more specific selectors will override more general ones. Pseudo-elements and pseudo-classes are counted as normal elements and classes, respectively.
  4. Finally, sort by order specified: if two declarations have the same weight, origin and specificity, the latter specified wins. Declarations in imported style sheets are considered to be before any declarations in the style sheet itself.

CSS Precedence

Precedence is another commonly used term for the algorythm CSS applies to resolve conflicting rules. Precedence is determined by the cascade and its key components - specificity and inheritance.

Specificity calculations

Specificity describes the weight of a selector based on the identifiers it contains. It is expressed as a comma-delimited group of four numbers. For each identifier in the selector, we increment the number in the respective position.

Specificity values are cumulative. If a selector contains two elements and three classes, its specificity would be 0,0,3,2.

Specificity values are sorted in right-to-left precedence. Thus a selector contianing 5 element identifiers (0,0,0,5) has a lower specificity than a selector containing just a single class (0,0,1,0).

Inheritance

In a document (XML, HTML), the elements form a treelike hierarchy, with the root element at the top of the tree and the rest of the elements spreading out below it.

In an HTML document, the html element is the root, with the head and body descending from it. The rest of the document structure descends from those elements.

CSS uses the document tree for the mechanism of inheritance - descendants inherit the style properties of their ancestors.

Some properties are not inherited, such as padding, margin, border and background. See this chart for a full list of inherited and non-inherited properties.

Text Formatting

Fonts

Font Families

  • serif - proportional font whose characters have serifs (decorations on the ends of strokes). E.g.: Times, Georgia, New Century Schoolbook.
  • sans-serif - proportional fonts with no serifs. E.g.: Arial, Helvetica, Verdana.
  • monospace - not proportional, emulate typewritten text. E.g. Courier, Courier New.
  • cursive - emulate human handwriting. E.g.: Comic Sans, Author.
  • fantasy - not defined by any single characteristic other than our inability to classify them in one of the other families. E.g.: Western, Woodblock, Klingon.

Layout

Boxes and display

There are two primary layouts defined by CSS - block-level and inline-level.

For the block-level layout, see this representation of the CSS box model.

The following rules apply to inline box: For the properties left, right, top, bottom, margin-left, margin-right, margin-top, and margin-bottom, any value of auto is converted to 0 (zero). width and height do not apply to nonreplaced inline boxes.

For replaced inline boxes (images, input elements, etc.), there are set of more obscure rules, defined here: http://www.w3.org/TR/css3-box/#inline-replaced.

Floating

  • float - left | right |none
  • clear - left | right | both |none

Positioning

The types of positioning are static, relative, absolute and fixed.

Static positioning

This is the default - the element box is generated as normal and is not taken out of the document flow.

Relative positioning

The element preserves the shape and space it would have occupied if it were not positioned, but it is offset by some distance.

Absolute positioning

The element is taken out of the flow and positioned with respect to its containing block. The space it would occupy if not positioned is closed up.

Fixed positioning

The element is absolutely positioned with respect to the viewport. When scrolling, it remains fixed.

  • left - <length> | <percentage> | auto
  • right - <length> | <percentage> | auto
  • top - <length> | <percentage> | auto
  • bottom - <length> | <percentage> | auto
